# QDRN Radar — Admin Runbook

For you (the operator). Everything the friend never needs to see.

## Access

- **Public app:** `https://radar.qdrn.io/md`
- **Setup wizard (friend, PIN-gated):** `https://radar.qdrn.io/md/setup`
- **Admin console (you, Cloudflare Access):** `https://radar.qdrn.io/md/admin`
- **Remote SSH:** `ssh pi@ssh.radar.qdrn.io` (via `cloudflared access`, see
  `infra/cloudflared/README.md`)

## .env reference

| Key | What |
|-----|------|
| `BASE_PATH` | URL base, default `/md` |
| `CF_TUNNEL_TOKEN` | Cloudflare tunnel token (token mode) |
| `RECEIVER_LAT/LON/CITY` | Map center (city-level) — also editable in the wizard |
| `FEEDER_LAT/LON/ALT` | **Exact** receiver location for the decoder/feeders |
| `RANGE_RINGS_NM` | Range ring radii, e.g. `50,100,150` |
| `SETUP_PIN` | PIN the friend types to open `/md/setup` |
| `ADMIN_EMAILS` | Comma-list allowed in the admin console (matched vs CF Access) |
| `ULTRAFEEDER_UUID` | Stable feeder UUID (auto-generated by install.sh) |
| `FR24_SHARING_KEY` / `PIAWARE_FEEDER_ID` | Feeder keys |
| `MAP_STYLE_URL` | MapLibre style JSON (default CARTO dark matter) |

## Common tasks

```bash
docker compose ps                 # status
docker compose logs -f qdrn-radar # app logs
docker compose restart qdrn-radar # restart just the app
docker compose pull && docker compose up -d   # update images
docker compose up -d --build qdrn-radar        # rebuild app after code changes
```

The admin console (`/md/admin`) does status / restart / log-tail from the browser
(it talks to the Docker socket mounted into the `qdrn-radar` container).

## Data & enrichment

- App data (settings, stats, enrichment cache) lives in `./data/qdrn-radar.db`
  (SQLite). Back this up to keep history.
- Enrichment uses free **adsbdb** + **hexdb** by default; if the friend adds a
  **FlightAware AeroAPI** key, routes/schedules upgrade automatically.
- Keys entered in the wizard are stored in the app DB. **Note:** wiring those
  keys into the `fr24feed`/`piaware` containers is currently via `.env`
  (`FR24_SHARING_KEY`, `PIAWARE_FEEDER_ID`) — see the roadmap to auto-apply them.

## Troubleshooting

- **No planes:** check the SDR is detected (`lsusb`), DVB driver is blacklisted
  (`/etc/modprobe.d/blacklist-rtlsdr.conf`), and `ultrafeeder` is healthy. Visit
  the raw map at `http://<pi>:8088` to confirm decoding.
- **Map blank:** the Pi (and your browser) need internet for the basemap tiles;
  self-host a style via `MAP_STYLE_URL` if you want full offline.
- **Admin 403:** confirm the Cloudflare Access policy includes your email and the
  `/md/admin` + `/md/admin/api` paths are both covered.
